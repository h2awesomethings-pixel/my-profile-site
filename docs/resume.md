# 이력서 — 정현화

## 자기소개

저는 맡은 일은 반드시 해낸다는 것을 원칙으로 삼는 개발자입니다. 명확한 원인을 찾기 어려운 문제도 포기하지 않고 전체 구조를 분석하며 실마리를 찾아왔습니다. 외부 연동 지연은 처리 흐름을 재검토해 비동기 구조로 개선했고, 응답이 느리던 주요 API는 인덱스 최적화를 통해 밀리초 단위로 개선했습니다.

문제를 마주하면 누군가 해결해주길 기다리기보다 직접 원인을 분석하고 해결하는 것을 당연하게 여깁니다. 회원 데이터 구조를 분석하다 발견한 정합성 이슈를 직접 개선했고, 외부 솔루션 의존 기능을 자체 시스템으로 전환할 때는 프로세스 설계부터 API 개발, 유관 부서 협의까지 주도했습니다.

결제·인증 도메인에서 대규모 트래픽을 다루며 성능 개선, 신규 서비스 구축, 아키텍처 개선을 두루 경험했습니다. 서비스 구조와 데이터 흐름을 깊이 이해하고, 운영 중 발생하는 문제를 스스로 찾아 해결하는 개발자로서 팀에 기여하겠습니다.

## 기술 스택

- **Backend**: Java, Spring Boot, Spring, RESTful API, Eureka, Config Server, API Gateway
- **Database**: MySQL, Oracle, MariaDB, Redis — 인덱스 설계·쿼리 튜닝
- **Infra & Tools**: Git, Linux, 보안 구성, 암호화
- **기타**: JavaScript(쇼핑몰 연동 SDK 개발), 결제·본인인증 연동, ISMS 심사 대응

## 경력

### 브이피주식회사 — 과장

**2023.04 ~ 현재 (3년 4개월)**

#### 서비스개발팀 (2026.03 ~ 현재)

##### 공통 서비스 재구축 및 이관 (2026.04 ~ 진행 중)

사업 구조 변화에 맞춰 공통 인증·연동 서비스를 신규 시스템으로 재구축하고 이관하는 프로젝트를 진행하고 있습니다. 서비스 인프라 구성, 공통 API 이관, 데이터베이스 전환, 보안 구성 개선을 함께 수행하며 안정적인 서비스 전환을 목표로 작업하고 있습니다.

##### 통신사 구독 서비스 개발 및 운영 (2026.03 ~ 진행 중)

외부 채널로 유통되는 구독 서비스의 신규 개발과 운영·유지보수를 담당하고 있습니다. 광고·제휴 채널 연동을 진행하고 있으며, 소규모 개발 인력의 업무 분배와 일정 관리를 맡아 프로젝트를 리드하고 있습니다.

##### 신규 PUSH 시스템 구축 (2026.03 ~ 진행 중)

외부 솔루션 기반으로 운영되던 PUSH의 성능·운영 이슈를 줄이고 자체 운영 역량을 확보하기 위해 신규 PUSH 시스템을 구축하고 있습니다. 설계부터 구현까지 단독으로 수행하며 다른 업무와 병행해 진행하고 있습니다.

#### 후후차세대TFT (2025.01 ~ 2026.02)

##### URL 탐지 시스템 자체 구축 (2025.11 ~ 2026.03)

외부 솔루션에 의존하던 URL 검증 기능을 자체 시스템으로 전환하는 프로젝트를 진행했습니다. 룰 기반 스코어링 방식의 탐지 프로세스를 설계했으며, 룰 정의 초안 제안부터 전체 프로세스 설계를 주도하고 URL 탐지 API 개발을 담당했습니다. 대규모 요청 환경에서 데이터 저장소 부하를 줄이기 위해 캐싱 구조를 적용했고, 관련 부서와의 연동 협의도 직접 진행했습니다.

전환 결과 탐지 품질과 운영 효율을 개선했고, 외부 솔루션 의존도를 낮추는 성과를 만들었습니다.

##### 아키텍처 구조 개선 (2025.09 ~ 2026.02)

보안 경계가 불명확했던 시스템 구조를 분석하고, 서비스 역할과 네트워크 구간을 분리하는 방향으로 개선했습니다. 기존 시스템 구조를 분석해 전환 설계의 근거를 마련하고 인프라팀과 협업하여 전환을 진행했으며, 서비스 간 통신 구조와 불필요한 연결 지점을 함께 정리했습니다.

이 과정에서 캐시 저장소와 애플리케이션 프레임워크 업그레이드를 여러 연관 프로젝트에 걸쳐 수행하며, 프로젝트별 의존성 변경과 버전 차이로 인한 데이터 직렬화 호환성 이슈를 해결했습니다. 개선 결과 보안성과 운영 안정성이 향상되었고, 만료 없이 누적되던 캐시 데이터에 TTL을 적용하여 불필요한 데이터 적재 문제도 함께 정리했습니다.

##### 사용자 참여 기능 역제안 및 미션 기능 구축 (2025.07 ~ 2025.10)

서비스 활성화를 위해 개발자 입장에서 기획을 역제안한 사례입니다. 여러 서비스의 사용자 참여 기능을 벤치마킹해 미션·래플·만보기와 서비스 내 콘텐츠를 활용한 참여형 기능 아이디어를 제안했습니다. 팀원이 제안한 친구초대와 함께 안건이 채택되어 기획·앱·서버 담당자가 함께 논의하는 협의체가 구성되었습니다.

협의체에서 진행 순서를 직접 제안했습니다. 친구초대의 필수 기반이면서 미션에도 활용할 수 있는 공통 기능인 '앱 공유하기'를 먼저 개발하도록 우선순위를 정리했고, 이어진 미션 기능 개발에서 서버 파트를 맡아 미션 목록 조회 API와 이를 운영하기 위한 어드민 기능(버전 체크, 미션 리스트 관리)을 설계·구축하여 출시했습니다.

##### 미사용 서비스 및 데이터 정리 (2025.04 ~ 2025.10)

시스템 전환을 앞두고 누적된 미사용 서비스·데이터 정리를 진행했습니다. 운영기획팀과 미사용 판단 기준을 정립하고, 전체 프로젝트 소스코드를 직접 분석해 실제 참조 여부를 확인하는 방식으로 제거 근거를 마련했으며, 서비스 영향을 최소화하는 절차로 정리 작업을 수행했습니다.

대량의 미사용 데이터와 종료된 서비스의 코드·연관 테이블을 정리하여 저장 공간 사용량을 줄이고 시스템 복잡도를 낮췄습니다.

##### 서비스 성능 개선 (2025.02 ~ 2025.06)

대용량 데이터를 다루는 API에서 발생하던 응답 지연을 원인 파악부터 튜닝까지 직접 수행했습니다. 단일 컬럼 인덱스를 각각 생성하여 실제로는 하나만 사용되던 문제와 인덱스 미적용으로 인한 풀스캔이 주요 원인이었으며, 복합 인덱스 적용과 인덱스 재설계로 개선했습니다.

- 피드 API: 데이터가 많은 특정 사용자에게 발생하던 타임아웃을 인덱스 최적화로 밀리초 단위 응답까지 개선
- 본인인증 검증 API: 전체 사용자 대상으로 상시 발생하던 지연을 인덱스 재설계로 개선
- 메모 API: 특정 사용자에게 발생하던 타임아웃을 쿼리 튜닝으로 개선

##### 회원 데이터 구조 개선 (2025.01 ~ 2025.04)

서비스 운영 중 발생하던 회원 데이터 문제를 분석하며 두 가지 이슈를 발견하고 직접 개선했습니다. 회원 식별 데이터 간 정합성 문제를 발견해 DBA와 협업하여 복구했으며, 탈회 실패 시 잘못된 기준값으로 조회를 이어가던 로직 오류를 정상화하여 불필요한 조회가 반복되던 문제를 해소했습니다.

두 작업 모두 원인 분석부터 코드 수정까지 직접 수행했으며, API 지연 감소·오류 감소·탈회 프로세스 정상화를 달성했습니다.

#### 인증결제팀 (2023.04 ~ 2024.12)

##### 웹 간편결제 ISP Pay 개발 (2024.07 ~ 2024.11)

결제 인증 도메인 경험을 바탕으로 웹 기반 간편결제 서비스를 신규 구축하는 프로젝트에 참여했습니다. 사전에 진행한 간편결제 프로토타입 개발 경험을 바탕으로 외부 결제 연동과 회원 구조를 설계에 반영했고, 서버 구조 설계 논의에 참여해 방향을 수립했습니다.

회원 관리, 카드 관리, 정기결제 프로세스 개발과 쇼핑몰 연동 JavaScript SDK 개발을 담당했으며, 보안 요건에 맞춘 암호화 초기화 구조와 Key 생성·검증 로직을 구현했습니다. 자사몰에 실제 서비스를 런칭했습니다.

##### 피싱케어 서비스 개발 (2024.02 ~ 2024.06)

기존 결제 앱을 기반으로 외부 채널을 통해 유통되는 구독형 서비스를 개발했습니다. 인증 프로세스를 새로 추가하고, 인증 결과와 기존 회원을 연결하는 구조를 설계·구현했으며, 기존 회원 구조에 맞춰 체크 로직을 전반적으로 재설계했습니다.

채널별로 상이한 화면·정책 요건은 페이지 분리로 대응했고, 대표 회원이 결제 후 구성원을 초대해 함께 이용하는 패밀리 기능도 도입했습니다. 현재까지 실서비스로 운영 중입니다.

##### 외부 결제 연동 안정성 개선 (2023.07 ~ 2023.08)

외부 결제 연동에서 간헐적으로 발생하던 지연을 분석했습니다. 명확한 원인 파악이 어려운 상황에서 전체 프로세스를 검토하던 중 비동기 처리가 가능한 알림 구간이 동기로 처리되고 있음을 발견했고, 비동기 구조 전환을 직접 제안하고 구현했습니다. 전환 이후 간헐적 지연 빈도가 줄어 인증 처리 안정성이 개선되었습니다.

### (주)큐브릿지 — 주임, 정보개발팀

**2018.12 ~ 2023.04 (4년 5개월)**

#### 커머스 핵심 기능 및 보안 고도화

외부 시스템에 의존하던 포인트를 자체 포인트 시스템으로 전환하기 위해 기존 프로세스를 분석하고 적립·사용·소멸 프로세스를 설계·구현했습니다. 커머스 운영에 필요한 외부 시스템을 API 방식으로 연동하여 주문·배송 정보 흐름을 지원했습니다.

ISMS 인증 심사 대응을 위해 2FA(SMS 인증) 로그인, 로그인 실패 제한 정책, 개인정보 접근 권한 관리 및 행위 이력 기능을 어드민에 구현했습니다.

#### 물류·재고(WMS) 시스템 구축 및 개선

물류 서비스 전환 프로젝트에 참여하여 상품관리, 재고관리, 창고관리, 배송조회 등 물류 프로세스 전반의 연동을 담당했습니다. 운영 중 WMS(실사 기반)와 자체 시스템(주문 기반)의 재고 집계 방식 차이와 배치 실행 시간 차이로 발생하던 데이터 불일치를 발견하고, 연계 로직을 정비하여 데이터 정합성을 확보하고 수동 확인 업무를 제거했습니다.

#### 검색엔진 운영 및 개선

데이터 증가로 인한 인덱싱 지연과 검색 오류를 분석하여 풀/증분 인덱싱 쿼리를 최적화하고, API 오류 수정과 처리 로직 개선으로 검색 인덱싱 안정성을 확보했습니다.
